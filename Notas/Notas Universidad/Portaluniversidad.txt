Portal universidad

Test está conformado por 4 VMs, 2 corresponden al código fuente de wordpress(web), otra a la DB y la última al GFS.

DB - Redis
La VM de la DB y redis tiene mysql version 8.0.36, la base de datos tiene las mismas credenciales que tenía el sitio, por el momento no se configuró ninguna directiva extra. Crear usuario y base de datos.
Solo para test, en la tabla options cambiar la url para evitar inconsistencias.
Crear el usuario para cada IP que vaya a acceder y otorgar permisos al usuario: 
CREATE USER 'myuser'@'%' IDENTIFIED BY PASSWORD '*HASH';
GRANT ALL ON mydb.* TO 'myuser'@'%';

Probar con:
mysql -h xxx.xx.xx.xx -u c1_www -p
FLUSH PRIVILEGES;

Configurar y comparar el archivo redis.conf (Importante!! Agregar la IP correspondiente a la máquina donde se encuentra instalado el redis)
bind 127.0.0.1 xxx.xx.xx.xx

Se pueden monitorear las conexiones entrantes a redis con redis-cli, es necesario instalarlo:
apt-get install redis-cli
redis-cli
MONITOR
redis-cli -h xxx.xx.xx.xx -p 6379



GFS
La máquina donde se encuentra el GFS cuenta con la configuración del protocolo nfs que permite compartir directorios/archivos entre diferentes máquinas dentro de la misma red. 
Instalar y configurar nfs:
apt-get install nfs-kernel-server nfs-common -y
En caso de ser necesario, crear la carpeta compartida
	mkdir /var/nfsshare
Agregar directorio, directivas y las IPs que van a acceder a los archivos compartidos:
nano /etc/exports 
		/var/nfsshare    xxx.xx.xx.xx(rw,sync,no_root_squash,no_all_squash)
	
systemctl restart nfs-kernel-server
	systemctl enable nfs-kernel-server

Migrar la carpeta wp-content en  /var/nfsshare

Otorgar permisos necesarios: 
	 chown -R www-data:www-data wp-content

Web
En las VMs de los web, además del código fuente, se encuentra php8.1, nginx y el cliente nfs.
	apt-get install php8.1 nginx
	apt-get install nfs-common -y
	apt-get install php8.1-fpm php8.1-gd  php8.1-imap php8.1-intl php8.1-mbstring php8.1-mysql php8.1-opcache php8.1-pspell php8.1-readline php8.1-soap php8.1-sqlite3 php8.1-tidy php8.1-xml php8.1-xmlrpc php8.1-xsl php8.1-zip
	apt-get install php-apcu php-bz2 php-common php-google-recaptcha php-imagick php-memcache php-mysql php-pear php-phpmyadmin-motranslator php-phpmyadmin-shapefile
apt-get install php-phpmyadmin-sql-parser php-phpseclib php-psr-cache php-psr-container php-psr-log php-soap php-symfony-cache php-symfony-cache-contracts php-symfony-expression-language

Configurar límites máximos en php:
	nano /etc/php/8.1/fpm/php.ini
post_max_size = 50M
upload_max_filesize = 50M
Configurar nginx:
	nano nginx.conf
Crear virtual server y ssl en nginx:
	nano /etc/nginx/sites-enabled/default
Configurar el nfs:
	nano /etc/fstab
	xxx.xx.xx.xx:/var/nfsshare             /mnt    nfs     nfsvers=3,defaults        0 0
Montar la carpeta:
	mount -a
Migrar la carpeta web de wordpress excepto wp-content
Crear link simbólico para la carpeta wp-content apuntada a la carpeta compartida:
	cd /../web/
	ln -s /mnt/wp-content/ wp-content
Editar el archivo .user.ini modificando la ruta de wp-content
Editar el archivo wp-config de wordpress y eliminar/renombrar .htaccess
Acceder al sitio y borrar caché
