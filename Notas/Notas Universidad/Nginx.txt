Crear un proxy con Nginx en Ubuntu :
 
Paso 1: Instalación de Nginx

    Actualizar los paquetes del sistema:

sudo apt update
sudo apt upgrade

Instalar Nginx:
    sudo apt install nginx

Paso 2: Configuración del Proxy

    Crear un archivo de configuración para tu proxy:

    Puedes crear un archivo de configuración en el directorio /etc/nginx/sites-available/. Por ejemplo, crea un archivo llamado mi-proxy:

sudo nano /etc/nginx/sites-available/mi-proxy

Agregar la configuración del proxy:

nginx

server {
    listen 80;

    server_name tu_dominio.com;

    location / {
        proxy_pass http://127.0.0.1:3000;  # Reemplaza con la URL de tu aplicación backend
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

Guarda y cierra el archivo.

Crear un enlace simbólico en el directorio sites-enabled:

sudo ln -s /etc/nginx/sites-available/mi-proxy /etc/nginx/sites-enabled/

Probar la configuración de Nginx:

sudo nginx -t

Reiniciar Nginx para aplicar los cambios:

    sudo systemctl restart nginx

Paso 3: Configuración Adicional

Si estás utilizando HTTPS, necesitarás configurar SSL en tu servidor Nginx. Aquí tienes un ejemplo de cómo hacerlo utilizando Let's Encrypt:

    Instalar Certbot y el complemento de Nginx:

sudo apt install certbot python3-certbot-nginx

Obtener el certificado SSL:

    sudo certbot --nginx -d tu_dominio.com -d www.tu_dominio.com

    Seguir las instrucciones en pantalla para completar la instalación del certificado SSL.

Paso 4: Verificación

Finalmente, verifica que tu configuración funciona correctamente accediendo a tu dominio en un navegador web. Deberías ver que las solicitudes se están proxy pasando correctamente a tu backend.
Resumen

    Instala Nginx.
    Crea y configura un archivo de configuración en /etc/nginx/sites-available/.
    Crea un enlace simbólico en /etc/nginx/sites-enabled/.
    Prueba la configuración y reinicia Nginx.
    (Opcional) Configura SSL con Certbot.

Siguiendo estos pasos, deberías tener un proxy inverso funcional utilizando Nginx en Ubuntu.

-----

En el paso 2, la IP que debes especificar en la directiva proxy_pass es la IP o el hostname del servidor backend al que deseas redirigir el tráfico. No es la IP del dominio del proxy, sino la dirección del servidor donde tu aplicación backend está ejecutándose.

Por ejemplo, si tu aplicación backend está ejecutándose en un servidor local en el puerto 3000, usarías http://127.0.0.1:3000. Si tu backend está en otro servidor, deberías usar la IP o el dominio de ese servidor. Aquí tienes un ejemplo más claro:
Ejemplos:

    Aplicación backend en el mismo servidor que Nginx:

    nginx

location / {
    proxy_pass http://127.0.0.1:3000;  # La aplicación backend está en el mismo servidor, en el puerto 3000
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
}

Aplicación backend en un servidor remoto:
Supongamos que tu backend está en un servidor con la IP xxxx.xxxx.xxx y en el puerto 4000:

nginx

location / {
    proxy_pass http://xxxx.xxxx.xxx:4000;  # La aplicación backend está en un servidor remoto, en el puerto 4000
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
}
