Funciona en NFS - DEV WORKER 1

Para configurar Postfix en Ubuntu y que tu sistema operativo pueda enviarte correos electróusuarios, sigue estos pasos:

    Instalar Postfix:
    Abre una terminal y ejecuta:

    bash

sudo apt update
sudo apt install postfix mailutils

Configurar Postfix:
Durante la instalación, se te pedirá que configures Postfix. Selecciona "Sitio de Internet" y luego ingresa el nombre de dominio de tu servidor o deja el valor predeterminado.

Configurar archivo main.cf:
Abre el archivo de configuración de Postfix:

bash

sudo nano /etc/postfix/main.cf

Asegúrate de tener las siguientes configuraciones (ajusta según tus necesidades):

Comentar TLS y snakeoil

plaintext
smtpd_relay_restrictions = permit_mynetworks permit_sasl_authenticated defer_unauth_destination
myhostname = dev-univ.universidad.edu.ar
alias_maps = hash:/etc/aliases
alias_database = hash:/etc/aliases
myorigin = /etc/mailname
mydestination = $myhostname, mail.universidad.edu.ar, eei-dev-worker, localhost.localdomain, localhost
relayhost = mail.universidad.edu.ar:25
mynetworks = 127.0.0.0/8 [::ffff:127.0.0.0]/104 [::1]/128
mailbox_size_limit = 0
recipient_delimiter = +
inet_interfaces = all
inet_protocols = all

smtp_use_tls=yes
smtp_sasl_auth_enable = yes
smtp_sasl_password_maps = hash:/etc/postfix/sasl_passwd
smtp_sasl_security_options = noanonymous

smtp_tls_CAfile = /etc/ssl/certs/ca-certificates.crt
smtp_generic_maps = hash:/etc/postfix/generic

Configurar autenticación SMTP:

touch  /etc/postfix/sasl_passwd
nano /etc/postfix/sasl_passwd


Añade la siguiente línea (sustituyendo tu_email@gmail.com y tu_contraseña con tus credenciales reales):

[smtp.gmail.com]:587 tu_email@gmail.com:tu_contraseña

Nota: Si usas Gmail, es posible que necesites generar una contraseña de aplicación en lugar de usar tu contraseña habitual de Gmail.

Establecer permisos y generar el archivo hash:

bash

sudo chmod 600 /etc/postfix/sasl_passwd
sudo postmap /etc/postfix/sasl_passwd

Reiniciar Postfix:

bash

sudo systemctl restart postfix

Probar la configuración:
Intenta enviar un correo electróusuario de prueba para asegurarte de que todo está funcionando correctamente:

bash

    echo "Este es un correo de prueba" | mail -s "Prueba de Postfix" tu_destinatario@dominio.com

# Directorio donde se encuentran los certificados de Let's Encrypt
CERT_DIR="/var/www/ojs.localidad-conicet.gov.ar/ssl"

# Archivo para almacenar los hashes de los certificados
HASH_FILE="/var/www/ojs.localidad-conicet.gov.ar/private/hash.txt"

# Archivo temporal para almacenar los hashes actuales
TEMP_HASH_FILE="/var/www/ojs.localidad-conicet.gov.ar/private/hash_actual.txt"

# Archivo de log para registrar los cambios
LOG_FILE="/var/www/ojs.localidad-conicet.gov.ar/private/certificate_changes.log"

CERT_ARCHIVE="/var/www/ojs.localidad-conicet.gov.ar/ssl/certificados.tar.gz"

# Dirección de correo del destinatario
EMAIL="usuario.apell@universidad.edu.ar"
##EMAILCC3="nusuario@universidad.edu.ar"
##EMAILCC4="soporte@universidad.edu.ar"

# Generar los hashes actuales de los certificados
find "$CERT_DIR" -type f \( -name "*.crt" -o -name "*.pem" -o -name "*.key" \) -exec sha256sum {} \; > "$TEMP_HASH_FILE"

# Recuperar las fechas del nuevo certificado
FECHASSL=$(openssl x509 -dates -noout < /var/www/ojs.localidad-conicet.gov.ar/ssl/www.esociales.universidad.edu.ar-le.crt)

# Comparar los hashes actuales con los anteriores
if ! diff "$HASH_FILE" "$TEMP_HASH_FILE" > /dev/null; then
    # Si hay cambios, registrar los cambios en el log y enviar un correo
    echo "Se detectaron cambios en los certificados SSL el $(date)" >> "$LOG_FILE"

    # Comprimir los certificados y claves privadas
    tar -czf "$CERT_ARCHIVE" -C "$CERT_DIR" .

    # Enviar correo con mutt
    echo "Se detectaron cambios en los certificados SSL del sitio ojs.localidad-conicet.gov.ar IP: xxx.xx.xxx.xx el $(date). Fechas del nuevo certificado: $FECHASSL" | mutt -s "Cambios en certificados SSL>

    # Actualizar el archivo de hashes anteriores
    cp "$TEMP_HASH_FILE" "$HASH_FILE"
fi

# Limpiar el archivo temporal
rm "$TEMP_HASH_FILE"

# Borrar el archivo comprimido si existe
if [ -f "$CERT_ARCHIVE" ]; then
    rm "$CERT_ARCHIVE"
fi

Instalar NFS: 
-----------------------------------

https://sysadmins.co.za/setup-a-nfs-server-on-ubuntu/

-- Lado del servidor --

1- apt install nfs-common -y

2- sudo apt-get install nfs-kernel-server nfs-common -y

3- mkdir /vol
chown -R nobody:nogroup /vol

5- Escribir en el archivo /etc/exports ej.
echo '/vol    10.0.9.0(rw,sync,no_root_squash,no_all_squash)' >> /etc/exports

6- sudo systemctl restart nfs-kernel-server
   
7- sudo systemctl enable nfs-kernel-server

-- Lado del cliente --
 
1- sudo apt-get install nfs-common -y

2- sudo mount IP.DEL.SERVIDOR.NFS:/vol /mnt
   sudo umount /mnt
   df -h

3- Escribie en el archivo /etc/fstab
   sudo bash -c "echo 'IP.DEL.SERVIDOR.NFS:/vol /mnt nfs auto,nofail,noatime,nolock,intr,tcp,actimeo=1800 0 0' >> /etc/fstab"
   sudo mount -a
   df -h
   
4- Comprobar con un mkdir si esta compartida la carpeta /vol del servidor con las carpetas /mnt de los clientes

((Si no funciona utilizar:
 mount -vvvv -t nfs -o vers=3 IP.NFS.SRVR:/vol /mnt ))
