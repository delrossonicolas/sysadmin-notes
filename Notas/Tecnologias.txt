----RAID----

RAID: Reduntant Array Independent Disks

Agrupa varios discos, como si fuera uno
Seguridad, si falla un disco no perdemos toda la informacion
Velocidad, ssd rapidas
Un raid NO es un backup

Cuando trozeamos un bloque de informacion en el Raid, se guardan en los distintos discos en pedazos de xMB

Si un disco falla, hay otro que sale, los datos son copiados en todos los discos

Raid 0
No hay tolerancia a falla, los datos no son duplicados. Son DISTRIBUIDOS, la data es perdida cuando se cae un disco
No deberia ser llamado raid ni siquiera
Los bloques de 2 discos escriben la mitad de informacion en cada uno. Si perdemos un solo raid, perdemos todo.
Si hay un raid de 4 discos y perdemos uno, perdemos TODO
+Velocidad

Raid 1
Toreltante a fallas, la data es copiada en varios discos, cuando uno falla, el otro tiene un duplicado. 
Es el famoso MIRROR / ESPEJO
Si falla un disco, se retira y el sistema vuelve a copiar.
La lectura es mas rapida porque lee la mitad de los bloques de cada disco, mas discos es mas velocidad. Pero perdemos almacenamiento ya que solo se usa la mitad

Raid 5
Necesita 3 o mas discos, es comumente la mas usada. No es data duplicada, es data distribuida entre todos los discos, esta data llamada parity se usa para reconstruir la data en caso de perdida
Almacena una paridad que puede reconstruir la data perdida, pero la almacena en todos los discos
Si hay 5TB entre los 5 discos por ejemplo, hay 4TB disponibles, ya que el equivalente a un disco entero, se usa para guardar parity

Raid 10
Combina raid 1 y raid 0
Es un raid, dentro de otro raid, la data es distribuida entre dos, luego esos dos duplican la data en cada uno

----ZFS----

Todos los discos se combinan para crear un gran storage pool, bajo el gran pool no es nada mas que carpetas, compartiendo espacio

Mirrors: Dos o mas discos con contenidos compartidos - Mirror es Raid 1

Sistema de archivos, estable y solido

Se puede poner los dispositivos juntos creando un pool llamado Zpool

Se puede crear Raidz y mirror.

Hay que pensar que no esta manteniendo la data, esta guardando tiempo para que reemplaces el disco

El sistema de archivos es parte del zpool, puede tener propiedades heredadas. Por ejemplo todo dentro de /var tiene compresion

Que es vdev

----KeepAlived----

Keepalived es una herramienta de software que se utiliza para implementar el VRRP en una red. 
Funciona enviando mensajes entre los routers que participan en el VRRP para determinar quién es el router activo y quién es el router de respaldo. 
Si el router activo falla o se desconecta, el router de respaldo toma el control y comienza a enrutar el tráfico de la red.
